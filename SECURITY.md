# Security Policy

## Supported Version

הפרויקט מתוחזק מהענף `main`. אין כרגע התחייבות לתמיכה בגרסאות ישנות או SLA
לתיקוני אבטחה.

## Reporting A Vulnerability

אין לפתוח Issue ציבורי עבור חולשה או חשד לדליפת מידע.

דווחו באופן פרטי דרך:

`https://github.com/tzvikahe-ops/tachles-open-source/security/advisories/new`

יש לכלול:

- תיאור ההשפעה;
- שלבי שחזור מינימליים;
- הקבצים או נקודות הקצה הרלוונטיים;
- הצעה לתיקון, אם קיימת;
- האם ידוע על ניצול פעיל.

נשתדל לאשר קבלת דיווח בתוך 7 ימים. זמני תיקון תלויים בחומרה ובזמינות
המתחזקים. אין לבדוק חולשות מול משתמשים, מידע או תשתית שאינם בבעלותכם.

## Secrets

אם מפתח, token או credential נחשף, יש לבטל או לסובב אותו מיד. הסרתו מ-commit
חדש אינה מסירה אותו מהיסטוריית Git.

`TELEGRAM_WEBHOOK_SECRET` ו-`DISPATCH_SECRET` חייבים להיות מוגדרים בפריסה.
הפונקציות המוגנות מחזירות שגיאת תצורה כאשר ה-secret המתאים חסר.
